Q1. Gemini의 답변이 대화창 UI보다 클 경우 어떻게 처리하나요?
이 프로젝트에서는 긴 답변을 페이지네이션(Pagination) 방식을 통해 효율적으로 처리하고 있습니다.

자동 분할 로직: lib/vnDialogPagination.ts에 정의된 로직에 따라, AI의 답변이 설정된 글자 수(기본 160자)나 줄 수(기본 3줄)를 초과하면 여러 개의 '페이지'로 자동으로 나뉩니다.
안전한 자르기: 단순히 글자 수로 자르는 것이 아니라, 문장 부호(마침표, 느낌표 등)나 줄바꿈을 기준으로 자연스럽게 끊으며, 특히 <img=...>와 같은 특수 태그 중간에서 대화가 잘리지 않도록 보호합니다.
UI 조작:
비주얼 노벨 모드: 대화창 우측 하단에 페이지 번호와 '다음' 버튼이 나타납니다. 사용자는 대화창을 클릭하거나 Space, Enter, 오른쪽 화살표 키를 눌러 다음 내용을 확인할 수 있습니다.
스크롤 지원: 만약 내용이 한 페이지 내에서 미세하게 넘칠 경우를 대비해 대화창 내부에는 세로 스크롤(overflow-y-auto) 기능도 포함되어 있습니다.
Q2. 대화창 UI를 바꾸면 채팅창에 그대로 적용되나요?
네, 실시간으로 즉시 적용됩니다.

반응형 스타일링: 대화창의 테마(배경색, 투명도, 테두리, 글자색 등)는 interfaceConfig.uiTheme 데이터와 연결되어 있습니다.
렌더링 방식: ChatDialoguePanel.tsx 컴포넌트는 캐릭터 정보를 프롭(Props)으로 전달받아 렌더링될 때마다 buildVnDialogueStyles 함수를 통해 실시간으로 CSS 스타일을 생성합니다.
데이터 동기화: 생성 화면에서 UI 설정을 변경하고 저장하면 로컬 저장소(IndexedDB)의 캐릭터 데이터가 업데이트되며, 이를 사용하는 채팅 페이지가 변경된 값을 감지하여 UI를 다시 그립니다.
요약하자면, 긴 대화는 책장을 넘기듯 나누어 보여주어 가독성을 높였으며, UI 디자인 변경 사항은 즉각적으로 채팅 화면에 반영되도록 설계되어 있습니다. 추가로 궁금한 점이 있으시면 언제든 물어봐 주세요!
